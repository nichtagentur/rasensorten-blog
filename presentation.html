AI Content Automation

Wie ein Blog sich selbst baut

Stefan Machinek | Februar 2026

Das Problem

  • Content-Erstellung ist zeitintensiv
  • SEO erfordert Expertise + absolute Konsistenz
  • Skalierung braucht normalerweise Personal
  • Qualitaet sinkt, wenn Quantitaet steigt
"Jeder Gaertner kann einen Rasen anlegen. Aber wer pflegt 17 Rasenflaechen gleichzeitig?"

Die Vision

  • Vollautomatisierter Blog ohne manuellen Eingriff
  • Rules-based statt manuell
  • Jede Seite folgt demselben Algorithmus
  • Ergebnis: Lighthouse 100 / 100 / 100

17 Seiten in unter 1 Stunde -- reproduzierbar fuer jedes Thema

Architektur: 3 Silos, 17 Seiten, 1 Regelsystem

Der Content-Prozess: 5 Schritte

1. RESEARCH
→
2. GENERATE
→
3. CRITIQUE
→
4. IMPROVE
→
5. PUBLISH

Kein manueller Eingriff zwischen den Schritten.
Der Algorithmus entscheidet, wann Qualitaet erreicht ist.

Schritt 1: Research

  • Parallele AI-Agenten durchsuchen das Web
  • Botanische Fakten, RSM-Standards, Naehrstoffwerte
  • Wettbewerber-Analyse: Was rankt, was fehlt?
  • Search Intent pro Keyword identifizieren

Ergebnis: Strukturierte Wissensbasis -- kein Raten, sondern Daten.

Schritt 2: Generate

Template-DNA

  • Einheitliches HTML-Template
  • Inline CSS (~3KB)
  • Inline SVG pro Seite
  • JSON-LD Schema
  • 800-1200 Worte Content

Interlinking per Matrix

  • Keine manuellen Links
  • Regelsystem bestimmt Verlinkung
  • Bidirektional: A verlinkt B, B verlinkt A
  • Cross-Silo + Intra-Silo
"Das Template ist die DNA. Jede Seite ist ein Organismus, der nach demselben Bauplan waechst."

Schritt 3: Critique -- Die Checkliste

  • H1 unique + keyword-optimiert?
  • Meta Description 150-160 Zeichen?
  • Heading-Hierarchie ohne Spruenge?
  • Alle Matrix-Links vorhanden?
  • Inhalt faktisch korrekt?
  • SVG hat role="img" + aria-label?
  • Seitengroesse < 15KB?
  • Search Intent beantwortet?
  • Kein Marketing-Fluff?
  • Schema.org korrekt?

Die AI kritisiert sich selbst -- gegen messbare Kriterien, nicht gegen Gefuehl.

Schritt 4+5: Improve & Publish

Improve

Schwaechen aus Critique fixen. Fehlende Links ergaenzen. Fakten korrigieren.

Minify

HTML komprimieren. Whitespace entfernen. Dateien auf 6-17 KB.

Deploy

Git commit + push. GitHub Pages deployt automatisch. Live in Sekunden.

Vom Entwurf bis zur Live-Seite: kein menschlicher Zwischenschritt
